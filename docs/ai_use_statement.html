How AI Was Used
Workflow and interaction model

Workflow

AI was used through an iterative prompt-and-review loop inside VS Code via the Claude Code extension. The student described each deliverable in natural language, reviewed the generated output in a browser, then issued correction or refinement prompts. No generated artifact was accepted without review. The student controlled all design decisions — scope, style, content, and personas. AI acted as an implementation accelerator, not a designer.

💬
1. Describe
Student describes the artifact in natural language
🤖
2. Generate
Claude writes the HTML/CSS/SVG file and commits to git
🔍
3. Review & Refine
Student reviews in browser, requests changes or approves
Prompt Samples
Representative prompts used during the project (translated to English where original was in Romanian)
Student → Claude
"Create an MR headset prototype in HTML — landscape layout, dark AR aesthetic with scan lines, glassmorphism panels. 3 tabs: Hub (status + room controls), Rooms (room grid with toggle), Access (user management). Make it feel like a real AR HUD, not a generic dashboard."
Claude → Result
→ Generated prototype_mr2.html — 500+ lines, full AR HUD with backdrop-filter blur panels, scan-line overlay, corner bracket decorations, JavaScript tab navigation and room toggles.
Student → Claude
"The storyboard needs to be in this style" [attached reference screenshot of white-card UX storyboard with stick figures]. "6 panels, white cards, proper stick figures with head/eyes/smile, AR headset shown on figure."
Claude → Result
→ Recreated storyboard_ar.html from scratch — white-card style, SVG stick figures with circular heads, proper limbs, AR headset drawn as dark rect with cyan lenses, Lora italic captions.
Student → Claude
"Reduce the features of the MR headset, make it more compact, easier and more intuitive — it's too much content for a gaze-based AR interface."
Claude → Result
→ Simplified prototype from 5 complex tabs to 3 clean tabs. Removed nested sub-panels, increased touch target sizes, consolidated controls into larger room tiles.
Student → Claude
"Check that in the examples we use people from our persona profiles — not placeholder names."
Claude → Result
→ Identified "Alex Morgan" and "Sarah Morgan" as placeholders across 3 files. Replaced with Daniel Popa (USER) and Laura Marin (LIMITED) — actual project personas — in prototype_mr2.html, ar_sketches.html, and prototype_mobile.html.

Declaration of Academic Integrity

The use of AI tools in this project was fully intentional, acknowledged, and disclosed in accordance with course guidelines. AI was used to accelerate implementation of designs that were conceptualized, directed, and reviewed by the student team. All personas, project direction, feature decisions, and critical design choices were made by humans. AI did not replace the design thinking process — it executed it.

No AI-generated content was submitted as original human intellectual work without disclosure. This statement represents a complete and honest account of AI involvement in the INTELLIDEN project.
